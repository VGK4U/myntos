E2
Request Flow — Staff CRM Page Load
1
User browser hits mnrteam.com/staff/solar-leads → Replit reverse proxy → Node.js server on port 3000
2
server.js checks cookies for staff_token. No token → redirect to /login
3
Valid token → NDA enforcement injected into HTML. HTML served with no-cache headers
4
Browser renders page. JavaScript calls staffFetch() — auto-attaches staff_token cookie on every API call
5
API call hits FastAPI backend at /api/v1/crm/... on port 8000. Auth middleware validates token → looks up employee in DB
6
Backend queries PostgreSQL → returns JSON. Frontend renders table/cards. Files served from Object Storage at /storage/
E3
AI Calling Flow — Twilio + GPT-4
1
Staff triggers call from dashboard → POST /api/v1/ai-calling/outbound-call → Twilio REST API initiates call to lead's phone number
2
Lead answers → Twilio webhook fires to backend → TwiML response plays greeting (TTS, language auto-detected: Hindi / English)
3
Lead speaks → Twilio Speech-to-Text transcribes → sent to GPT-4 with full conversation history + knowledge base context
4
GPT-4 generates reply → backend TTS (Twilio) speaks response to lead. Loop continues until call ends
5
Call ends → recording stored → outcome tracked. Campaign auto-dials next lead. IVR entirely skipped for campaign calls
E4
Lead Document Share Flow
1
Staff clicks Share button on lead row → 2-phase modal opens. Staff chooses All or Selected doc types
2
POST /api/v1/crm/leads/{id}/share-link with optional doc_types array → backend generates 40-byte URL-safe token
3
Token stored in crm_lead_share_tokens table with expiry = now + 6 hours UTC + doc_filter JSON
4
Staff copies URL: mnrteam.com/lead-share?token=... and sends to client via WhatsApp/SMS
5
Client opens link → GET /api/v1/crm/share/{token} → backend validates expiry, filters docs by doc_filter → returns lead data
6
Share page renders: lead hero (Name + Lead ID + App No) · countdown timer · only included doc groups · View/Download per doc. No login needed. No edits possible

F — Security & Hosting Details
F1
Hosting & Infrastructure
Platform
Replit Cloud — managed cloud hosting on shared Linux containers (NixOS). Auto-restart on crash via loop wrappers in all workflow commands.
Domain
mnrteam.com — custom domain via Replit deployment (CNAME to Replit CDN). Dev environment also accessible at *.replit.app subdomain.
TLS / SSL
Enforced on all traffic. Replit provides automatic TLS certificates (Let's Encrypt). Preview pane uses Replit mTLS proxy. All HTTP → HTTPS redirect enforced.
Uptime
Processes managed by Replit Workflow engine. Backend: FastAPI with 4 Uvicorn workers. Frontend: Node.js single-process. Each restarts automatically within 3–5 seconds of crash.
Storage
PostgreSQL hosted by Replit (Neon-compatible). Replit Object Storage for all files. Both are persistent across restarts and deployments.
F2
Authentication & Access Control
Staff Login
Employee ID + password. Session token issued on success → stored as httpOnly cookie. All API calls require this token. Token validated on every request against DB.
NDA Gate
Every staff page has NDA enforcement injected server-side. Staff must accept NDA before first access. Acceptance recorded in nda_acceptances table with timestamp + IP.
Share Links
40-byte cryptographically random URL-safe tokens (secrets.token_urlsafe(40)). Hard expiry 6 hours from generation (UTC). Expired tokens return HTTP 410. Share page is read-only — no data can be modified by the recipient.
API Auth
All protected endpoints use FastAPI dependency injection to validate the session token before executing any query. Unauthenticated requests get HTTP 401.
CORS
Backend allows requests from the frontend proxy origin only. All API calls made via relative URLs to prevent cross-origin leakage.
F3
Data Security & Document Protection
At Rest
PostgreSQL data encrypted at rest by Replit infrastructure. Object Storage files encrypted at rest (AES-256 via Replit/Cloudflare R2).
In Transit
All traffic HTTPS/TLS 1.3. File download URLs served via the same authenticated session — not publicly guessable paths.
Documents
Files stored with randomised internal names (UUID-based). Access to /storage/{file} is only meaningful when the staff session is active or a valid share token is present.
Secrets
All API keys (Twilio, OpenAI, Meta, MyOperator) stored as Replit Secrets — never in code or config files. Injected as environment variables at runtime.
Badges
✓ TLS 1.3 ✓ AES-256 at rest ✓ httpOnly cookies ✓ Token auth ✓ 6h expiry links ✓ NDA enforcement ✓ Secrets vault
