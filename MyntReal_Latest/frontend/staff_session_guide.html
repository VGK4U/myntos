Staff Platform Features — March 2026 Session Guide

Three delivery streams: A — Day End Report Timesheet Fix  ·  B — Department KPIs  ·  C — Marketplace → Service Ticket Integration

What Was Delivered This Session
A Day End Report — Timesheet Fix

Fixed a real-world bug where staff who logged timesheets yesterday but haven't logged today would see "0h 0m Submitted" on their Day End Report. Now auto-falls back to the previous workday, shows a date note, and correctly labels hours as "Logged" vs "Submitted".

B Department KPIs in Day End Report

Each department now shows 3 personalised KPI tiles in the Day End Report and mobile progress card. Sales staff see talk time + leads; Procurement staff see parts received and pending; Service staff see tickets and TAT%. Hidden for all-other departments.

C Marketplace → Service Ticket Integration

Service staff can now pick spare parts directly from the VGK4U catalog while raising a spare request on any ticket. System auto-creates a ZYPO (in-stock) or ZYPR (out-of-stock) immediately. Source labels appear in procurement dashboards.

Day End Report — Timesheet Fallback + Label Fix
Affects: staff_progress.html (web) & ProgressPage.ts (mobile)
The Problem
Scenario: A sales executive submits timesheet entries on March 1st (4h 30m logged). On March 2nd morning (before logging anything), they click "Generate Day End Report". The report shows "Submitted: 0h 0m" because today (March 2) has no entries.

Same bug on mobile: ProgressPage showed the label "Submitted" even for ts_total_time_updated, which counts ALL entries regardless of submission status — the label was factually wrong.

Role-by-Role Operating Instructions
Sales Staff — Day End Report & KPI Tiles
How to Generate Your Day End Report
  1. Go to My Progress (today tab).
  2. Click Generate Day End Report.
  3. The report opens as a modal card with dark theme.
  4. If today's timesheet = 0h, the system auto-fetches yesterday's data and shows "(for DD Mon)" in amber. This is normal — it means your previous day's hours are shown.
  5. Look for the KPI tiles section: you'll see Talk Time, Leads Handled, and Overdue count.
  6. Use the Share button to send to WhatsApp or copy.
What the Labels Mean
LabelMeaning
LoggedAll time entries counted — whether draft, submitted, or approved. Shows when viewing historical dates or fallback data.
SubmittedTime entries formally submitted to manager. Shows only when viewing today's own data with actual submissions.
(for DD Mon)Amber note — report is using previous workday data because today has no entries yet.
Overdue (red tile)CRM leads whose next_followup_date has passed. Take action before closing the day.
Mobile ProgressPage

Open the mobile app → Today tab. Your progress card shows Logged / Approved hours (not just "Submitted"). The KPI row below shows Talk Time, Leads, Overdue inline. If dept_type = 'other', this row is hidden — nothing to worry about.

Service Manager — Spare Requests via Catalog
How to Request Spares Using the Catalog
  1. Open the service ticket in the Service Queue.
  2. Click Request Spares.
  3. Two buttons appear: Pick from Catalog (default, recommended) and Enter Manually.
  4. In Catalog mode: type part name or SKU in the search box. Wait ~400ms for results.
  5. Choose discount mode from the dropdown: MNR (3%), Dealer (12%), Student (10%), Partner (12%).
  6. Read the stock dot: In stock (→ ZYPO), Out of stock (→ ZYPR).
  7. Click a result to add to basket. Add multiple items if needed.
  8. If a part isn't in the catalog: switch to Enter Manually — type name, qty, estimated cost.
  9. Click Submit. System creates ZYPO/ZYPR automatically. Badges appear on spare rows.
ZYPO vs ZYPR — What Happens
ZYPO — Part is in stock
A Purchase Order is auto-raised against your company. No manual creation needed. Stock is NOT reduced yet — only when physically dispatched. ZYPO appears in the VGK4U PO list with Source = "Service Ticket #..."
ZYPR — Part out of stock
A Procurement Request is raised for the procurement team to source. If an existing open ZYPR exists for the same SKU, it's reused (no duplicates). ZYPR appears in Procurement dashboard with Source = "Service Ticket #..."
Rules to Remember
  • Do NOT create a separate ZYPO/ZYPR manually if you're using the catalog picker — it's already done.
  • Discount mode affects the price shown in the ZYPO/ZYPR document. Choose carefully.
  • If the part isn't in the VGK4U catalog (e.g., a third-party vendor part), use Manual mode.
  • Changing the spare after ZYPO/ZYPR creation requires cancelling the PO/PR first — contact Admin.
Procurement Staff — Viewing KPIs & Source-Linked Documents
Your Day End Report KPI Tiles

Your report shows 3 amber tiles: Received Today (count + ₹ value), Pending count, and Above TAT count. These come from the marketplace_procurement_requests table filtered to your assignments for that day.

Above TAT: Any PR older than 3 days still in pending/confirmed/ordered status. These need immediate vendor follow-up.
Finding Ticket-Sourced PRs in Dashboards
  1. Open VGK4U Procurement or Service Procurement page.
  2. Use the Source filter dropdown → select "Service Ticket" to see only ticket-sourced PRs.
  3. The Source column shows the linked ticket number as a clickable link.
  4. Click the ticket link to view full context: customer, vehicle, issue, why this part was needed.
Admin / RVZ — Full View & Troubleshooting
Day End Report Issues to Watch
  • If a staff member reports "my hours show yesterday's date" — that's the auto-fallback working correctly. Normal behaviour.
  • If KPI tiles don't appear → check their department name in HR master. Must contain "sales", "procurement", "service" or "purchase" to detect dept_type.
  • Talk time not showing → verify call logs are synced for that date.
Marketplace Source Tracking
  • All ticket-sourced ZYPOs have source_type = 'service_ticket' and source_ticket_id set.
  • Manually-created ZYPOs have source_type = 'public' (default).
  • Source column is searchable/filterable on all procurement dashboards.
  • Cancelling a ZYPO linked to a ticket → manually update the spare request's marketplace_po_id to null via Admin panel.
