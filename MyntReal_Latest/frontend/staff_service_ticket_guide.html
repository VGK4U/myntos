What is the Service Ticket Module?

The VGK4U EV Service Ticket system manages the full lifecycle of customer service requests — from initial complaint to resolution and billing. It handles two core types of issues: spares-based (physical part replacement) and technical (diagnosis, software, electrical).

Ticket Types
Spares
Requires physical parts (battery, brake pad, controller, motor, wiring harness). Triggers procurement pipeline — ZYPO or ZYPR is auto-created when spares are picked from catalog.
Technical
No physical part needed. Covers software flashes, motor tuning, BMS calibration, electrical faults, firmware upgrades. Goes directly to Ready for Work after diagnosis.
General
Generic support queries, documentation help, warranty claims, general complaints. Simpler workflow — no spares sub-pipeline.
How Tickets Enter the System
Website / Public
Customer submits from mnrteam.com. Rate-limited to 5/hr per IP. No login needed. Source = website.
Partner Portal
Dealer/Service Center raises on behalf of their customer. Partner JWT required. Source = partner_portal. Ticket is auto-linked to that partner.
Staff / Internal
Staff creates via Service Queue page. Can set ticket_type, assign technician, set priority. Source = website or phone.

TAT & SLA System
Base TAT
Default = 24 working hours from creation. Stored as tat_base_hours. A deadline (tat_due_at) is calculated and tracked.
Extension for Spares
Each spare requested adds +48 hours (tat_extension_hours on spare request row). Total TAT = base + sum of all extensions.
SLA Breach: If tat_due_at is passed without work_complete, sla_status flips from Within SLA → SLA Breached → automatic escalation to reporting manager.
Spares Ticket — Used when the EV needs a physical part replacement. This triggers the full procurement pipeline (ZYPO or ZYPR) and adds TAT extension per spare.

When a spare part is requested on a ticket, the system automatically creates either a Purchase Order (ZYPO) or a Procurement Request (ZYPR) based on stock availability. No manual step needed.
Decision Logic (WVV Protocol)
ZYPO — In Stock Path

Condition: available_qty ≥ quantity_required

Action: System creates a MarketplacePurchaseOrder with:
  • source_type = 'service_ticket'
  • source_ticket_id = ticket.id
  • PO number auto-generated: ZYPO-{year}-{seq}
  • Line item added with pricing

Stock NOT decremented at creation — only at physical dispatch.
Spare request: marketplace_po_id populated.
ZYPR — Out of Stock Path

Condition: available_qty < quantity_required

Action: System checks for an existing open ZYPR for the same SKU + company. If none → creates MarketplaceProcurementRequest with:
  • source_type = 'service_ticket'
  • triggered_by = 'service_ticket'
  • PR number: ZYPR-{year}-{seq}

Spare request: marketplace_procurement_id populated.
Pricing Calculation
Dealer Price
₹18,500 (base catalog price)
Discount Mode
MNR = 3%  |  Student = 10%  |  Dealer/Partner = 12%
Net Before Tax
dealer_price × (1 − disc%) → ₹18,500 × 0.97 = ₹17,945
GST Amount
net × (gst_pct / 100) → ₹17,945 × 0.18 = ₹3,230.10
Final Price
net + gst → ₹21,175.10
No margin floor check — Pricing is purely formula-driven. Margin floor (20%) applies only to category config pricing, not ticket spares.

Role-by-Role Operating Instructions
Service Manager
Full operational control of ticket lifecycle

Daily Checklist

1. Check New Tickets
Open Service Queue → filter by sub_status = new. Any ticket older than 2 hours without acknowledgment = priority action.
2. Acknowledge & Assign
Click ticket → ✓ Acknowledge → Select technician from dropdown → Confirm. SMS sent to customer.
3. Review Spares Pending
Filter by awaiting_spares. For each: check ZYPO/ZYPR status. If vendor delayed → update expected delivery date.
4. Release Spares
When parts physically arrive → click spare row → Release Spare. When ALL spares released → ticket auto-moves to Ready for Work.
5. Create Billing After Work Complete
Once technician marks work complete → open ticket → Create Bill → Enter amounts → Choose warranty status → Post.

SLA Watch Rules

Within 2 hrs of creation: Ticket must be acknowledged. If not, escalation auto-triggers.
Within 24 hrs: Diagnosis must be complete (for non-spares) or spares requested (for spares tickets).
Spares TAT: +48 hrs per spare. If 3 spares requested → up to +6 days extension possible. Communicate expected date to customer.
SLA Breach: If tat_due_at passes without work_complete → status shows red. Auto-escalation to reporting manager. You must add a comment explaining delay.

Staff Reassignment

If a technician is unavailable → Reassign Staff. Select new manager and/or technician. Reason required. Full history logged.

How to Raise a Ticket (Partner Portal)

  1. Log in to the Partner Portal at mnrteam.com/partner
  2. Go to Service Tickets → New Ticket
  3. Enter customer details (name, phone, vehicle model, serial)
  4. Select ticket type: Technical or Spares
  5. Describe the issue in detail (including error codes if any)
  6. Submit — ticket auto-links to your partner account

Viewing Your Charges

Go to Service Tickets → My Charges tab. Shows all bills raised against your service center including spares amount, labour, tax, payment status.

If you see a charge you disagree with → raise a query by commenting on the ticket. Do NOT raise a new ticket for billing disputes.

Media Upload Rules

  • Max 1 video (50MB)
  • Always upload defect photos before repair begins
